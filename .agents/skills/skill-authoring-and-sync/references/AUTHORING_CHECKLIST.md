# Authoring Checklist

## Scope

- Skill name is specific and unambiguous
- Description explains exact trigger conditions
- "When to use" and "when not to use" are both present

## Content Quality

- Steps are concrete and sequential
- No vague filler words without actionable meaning
- Includes practical examples and command snippets
- Includes anti-patterns or common mistakes

## Structure

- Frontmatter exists and is valid
- Sections are easy to scan
- References are linked and useful
- Output format/contract is defined when needed

## Validation

- Skill can be discovered by the local agent environment
- Public-facing docs are machine-neutral
- Local-only operational details are kept internal

## Sync

- MBP -> mini sync command runs successfully
- New/updated skill files exist on mini after sync
